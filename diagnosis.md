## 점검 1 (최우선) — GFC 구간 트랙 A / B 모순

### 원문

- **트랙 A (3-2):** "GFC(느린) 수익 +0.0%, MDD +0.0% — BIL/AGG 미상장으로 현금"  
- **트랙 B (4-1):** "2008 GFC  wd=OFF 수익 +12.0%, MDD −2.8%  → 중립(방어 이미 ON)"  
  + 주석: "전략이 이미 100% AGG(방어)였으므로 워치독 발동 없음"

### 증거

#### (1) AGG·BIL 실제 첫 거래일

| 종목 | 첫 거래일 | 비고 |
|---|---|---|
| AGG | **2003-09-29** | GFC(2007-10) 시작 전 존재 |
| BIL | **2007-05-30** | GFC 직전 상장 |

#### (2) 조건 B 계산 가능 시점

`signals.py:46` — `if len(monthly) < 13: return False`  
BIL 13번째 월말 = **2008-05-31** → 이 이전에는 조건 B 계산 불가 → `return False` (방어 처리)

즉, GFC가 시작된 **2007-10 시점에 BIL은 존재하지만 12개월 히스토리 미달** → 조건 B 불가.

#### (3) 조건 B 불가 시 코드 동작 (해당 라인)

```python
# signals.py:46-47
if len(monthly) < 13:
    return False  # 12개월 수익률 계산 불가 → 방어(보수)

# signals.py:54-55
if any(pd.isna(v) for v in (spy_now, spy_ago, bil_now, bil_ago)):
    return False  # BIL 미상장 등 데이터 공백 → 방어(보수)
```

`return False` → `_compute_phase_filter` = False → `compute_target_weights` 반환 = `{defense: 1.0}` = `{AGG: 1.0}`

이 `{AGG: 1.0}` 신호를 engine이 받는다. **AGG 가격이 있으면 AGG 매수 가능**하다.

#### (4) 2007-09 ~ 2009-04 월별 실제 보유 로그

| 날짜 | phase | 보유 | traded | pv_before | pv_after |
|---|---|---|---|---|---|
| 2007-09-04 | OFF | AGG | F | 1.000000 | 1.000000 |
| 2007-10-01 | OFF | AGG | F | 1.000000 | 1.000000 |
| … (중략) | | | | | |
| 2008-12-01 | OFF | AGG | F | 1.000000 | 1.000000 |
| 2009-01-02 | OFF | AGG | F | 1.000000 | 1.000000 |
| 2009-03-02 | OFF | AGG | F | 1.000000 | 1.000000 |

**traded=F (모두 False), pv=1.000000 고정**

#### (5) 자산 곡선 실측값

```
equity curve 2007-10-09 : 1.000000
equity curve 2009-03-09 : 1.000000  → 트랙 A 구간 수익 = 0.0000%  ✓

equity curve 2008-01-02 : 1.000000
equity curve 2009-12-31 : 1.119950  → 트랙 B 구간 수익 = +11.995%  ≈ +12%

2009-11-02 (첫 ON 매매) : 0.999750
2009-12-31              : 1.119950
→ 2009-11-02 → 2009-12-31 구간 수익 = +12.02%
```

### 진단

**트랙 A와 트랙 B는 둘 다 숫자는 맞다. 모순처럼 보이는 이유는 측정 창이 다르기 때문이다.**

| | 측정 기간 | SENTINEL 수익 | 원인 |
|---|---|---|---|
| 트랙 A | 2007-10-09 ~ 2009-03-09 (GFC 피크→저점) | +0.0% | 전구간 현금, 2009-03 전에 ON 없음 |
| 트랙 B | 2008-01-01 ~ 2009-12-31 (달력 연도) | +12.0% | **2009-11~12월 섹터 매수 수익** 포함 |

**실제 포트폴리오 상태:**  
1999-02 ~ 2009-10-30: `shares = {}` (현금). `target = {AGG: 1.0}` 이지만 **AGG를 한 번도 매수하지 않음**.

이유: 초기화(1999-02) 시 AGG가 미상장이어서 첫 매수 실패 → `shares={}, prev_target_key=frozenset({'AGG'})` 설정 → 이후 모든 월이 "신호 동일 → 무행동" → 2009-11 첫 ON 신호 변화 시에야 `_full_rebalance` 실행 (AGG→섹터 전환).

**리포트의 오류:**

| 항목 | 리포트 기술 | 실제 |
|---|---|---|
| 트랙 B 주석 | "전략이 이미 100% AGG(방어)" | **틀림** — 현금(shares={}) 상태 |
| 트랙 A 수익 | +0.0% | **맞음** — 현금이므로 0% |
| 트랙 B 수익 | +12.0% | **맞음** — 수치는 맞으나 출처가 잘못 설명됨 |
| +12%의 출처 | (미설명, AGG 보유로 추정 유도) | 2009-11~12월 섹터(XLE/XLK/XLY) 수익 |

**SPY Track B 수익 차이도 주의:**
- 트랙 A: SPY −55.2% (2007-10~2009-03 피크→저점)
- 트랙 B: SPY −19.4% (2008-01~2009-12, 2009 회복 포함)

두 수치 모두 계산은 정확하나 다른 창을 나타냄. 리포트 독자가 혼동할 수 있음.

## 점검 2 — 헤드라인 MDD가 다른 시기 비교

### 증거

```
전략 실제 작동 시작일: 2009-11-02 (첫 섹터 매수 = AGG → 섹터 전환)
```

| 지표 | 전체 구간(1999-02~) | 공정 비교 창(2009-11~) | SPY(공정 창) |
|---|---|---|---|
| **MDD** | **−27.4%** | **−27.4%** | **−33.7%** |
| CAGR | +6.0% | +10.0% | +14.7% |
| Sharpe | 0.54 | 0.70 | 0.89 |
| Sortino | 0.39 | 0.65 | 0.84 |

- 전체 구간의 SENTINEL MDD(−27.4%)는 2022-01 ~ 2023-10 발생
- 전체 구간의 SPY MDD(−55.2%)는 2007-10 ~ 2009-03 발생

```
공정 창(2009-11~) SENTINEL MDD: −27.4%  고점=2022-01-03  저점=2023-10-27
공정 창(2009-11~) SPY MDD     : −33.7%  고점=2020-02-19  저점=2020-03-23
```

### 진단

- **전체 구간 MDD 비교는 서로 다른 시기를 비교한다.** SENTINEL MDD는 2022-23 발생, SPY MDD는 2007-09 발생. 두 최악 구간이 겹치지 않는다.
- **그러나 공정 비교 창에서도 MDD 우위는 성립한다:** SENTINEL −27.4% < SPY −33.7% (+6.3%p).
- 전체 구간 CAGR(6.0%)은 초기 10년 현금 드래그를 포함해 낮아 보임. 공정 창에서는 +10.0%.
- 리포트가 이 차이를 명시하지 않은 것은 독자 오해를 유발할 수 있음.

**수치 오류 없음. 단, "다른 시기 MDD 비교"라는 한계를 리포트가 명시하지 않음.**

```

전환 후 AGG가 2022-03~10 사이 추가 −12.7% 하락
```

리포트 섹션 8의 "개선 필요 항목 1: 2022 방어력 미흡 → 방어 전환 타이밍"은 원인 진단이 틀림.

## 점검 4 — Sortino < Sharpe 원인

### 증거

#### SENTINEL 분모 비교

```
일별 수익률 수    : 6,874개
수익률=0인 날     : 2,710개 (39.4%) ← 현금 보유 기간 fill-forward

Sharpe 분모  = excess.std()           = 0.007606  (전체, N-1 sample std)
Sortino 분모 = sqrt(mean(neg²))       = 0.010448  (RMS of negative returns)
Sortino 분모 = neg_returns.std()      = 0.007769  (업계 표준 방식 시)

Sharpe  (코드)      = 0.54
Sortino (코드)      = 0.39  ← 리포트 기재값
Sortino (업계표준)  = 0.53  ← 코드 방식과 다름
```

#### SPY 분모 비교

```
수익률=0인 날     : 21개 (0.3%)

Sharpe 분모  = 0.012133
Sortino 분모(코드) = 0.012706

Sharpe  = 0.53
Sortino (코드) = 0.50
```

### 진단

**코드 버그 없음. 구현 방식의 컨벤션 차이.**

```python
# sentinel/metrics.py:131
downside_std = float(np.sqrt((downside ** 2).mean()))
```

이 구현은 음수 초과수익의 **제곱평균제곱근(RMS)**을 분모로 사용.  
업계 표준(예: QuantLib, PyPortfolioOpt)은 `downside.std()`(sample 표준편차)를 사용.

| 구현 방식 | SENTINEL 분모 | Sortino |
|---|---|---|
| 코드 (RMS) | 0.010448 | 0.39 |
| 업계 표준 (sample std) | 0.007769 | 0.53 |

`Sharpe > Sortino`가 발생하는 구조적 이유:  
- SENTINEL은 수익률=0인 날이 39.4%로 많음 (현금 기간 fill-forward)  
- 0인 날이 많으면 전체 std(Sharpe 분모)는 낮아짐  
- RMS(Sortino 분모)는 음수 날에만 계산하므로 0인 날의 영향을 받지 않음  
- 결과: Sharpe 분모 (0.007606) **<** Sortino 분모 (0.010448) → Sharpe > Sortino

**SPY도 같은 방향이지만 차이가 작은 이유:** SPY는 수익률=0인 날이 0.3%뿐이라 전체 std와 RMS의 차이가 작다.

**리포트의 "소르티노 역전 ⚠" 표기는 이상 징후를 올바르게 포착했으나,  
원인 설명이 없음. 실제 원인은 코드 구현 방식(RMS) + 현금 기간 inflate 효과.**
