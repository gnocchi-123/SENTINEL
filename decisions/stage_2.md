## 주요 설계 결정

### yfinance MultiIndex 처리
- yfinance 1.x: `yf.download([...])` 반환 컬럼이 `(Price, Ticker)` MultiIndex.
- `raw["Close"]`로 Close 레벨만 슬라이싱 → 티커명을 컬럼으로 갖는 일반 DataFrame 확보.
- 단일 티커가 Series로 반환되는 경우를 `isinstance` 분기로 처리.

### NaN 정책 — 채우지 않는다
- 늦은 상장 ETF의 상장 전 구간을 NaN 그대로 유지.
- 0 → NaN 강제 변환 미적용 (yfinance 기본값 신뢰, 향후 필요 시 재검토).
- 신호 계산 시 NaN은 자동으로 순위 풀에서 제외됨 (다음 단계에서 보장).

### 리샘플 alias — `"ME"`
- pandas 3.0에서 구 `"M"` alias 삭제됨 → `"ME"` (Month End) 사용.
- `resample("ME").last()` : 월별로 마지막 유효값(=마지막 거래일 종가) 선택.
- 결과 인덱스는 캘린더 월말 날짜 레이블(예: 2024-01-31). 모멘텀 계산 시 일관성 확보.

### 캐시 전략
- `price_data.csv` 존재 시 yfinance 미호출 → 오프라인 재현 보장.
- `--refresh` 플래그로 강제 갱신. `backtest.py` argparse에도 동일 플래그 추가.
- CSV 로드 시 `parse_dates=True` + `pd.to_datetime()` 이중 적용 → 인덱스 타입 보장.

### 타임존
- yfinance 1.x 일봉 인덱스는 tz-naive. 방어적으로 `tz is not None` 시 제거하는 코드 유지.

### `__main__` 블록
- `python sentinel/data.py [--params ...] [--refresh]`로 단독 실행 가능.
- 다운로드 + 무결성 리포트까지 자동 수행. 개발·검증 목적.
